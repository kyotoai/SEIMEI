# SEIMEI Patch Format

`seimei.editing.apply_patch` accepts payloads wrapped by:

```text
*** Begin Patch
...
*** End Patch
```

Only `*** Update File:` operations are supported.

## Default Format (EDIT Blocks)

This is the default format generated by `edit_file`.

```text
*** Begin Patch
*** Update File: relative/path.py
<EDIT insert=<line>>
<lines to insert before <line>>
</EDIT>
<EDIT replace=<start>-<end>>
<replacement lines for the deleted range>
</EDIT>
*** End Patch
```

Rules:
- Paths must be relative and stay inside the workspace.
- Line numbers are 1-based.
- All line numbers are interpreted against the original file before any edit block is applied.
- `<EDIT insert=<line>>` inserts text before `<line>` and must include at least one inserted line.
- `<EDIT replace=<start>-<end>>` replaces the inclusive range `<start>...<end>`.
- For delete-only, use `<EDIT replace=<start>-<end>>` with an empty body.
- Every edit block must end with `</EDIT>`.
- Edit ranges must not overlap, and insert positions cannot fall inside a replaced range.

Example:

```text
*** Begin Patch
*** Update File: README.md
<EDIT insert=1>
I'm in Kyoto
</EDIT>
<EDIT replace=1-3>
He is in Tokyo
</EDIT>
*** End Patch
```

## Fallback Formats

For compatibility, these formats are still accepted:
- Line-range hunks with `@@<line>` and `@@<start>-<end>`.
- Legacy Codex-style update hunks using context/`+`/`-` lines.

Different chunk styles cannot be mixed in the same `*** Update File:` block.
