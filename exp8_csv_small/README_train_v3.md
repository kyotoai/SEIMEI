# README_train_v3

## Overview

`exp8_csv_small/train_v3.py` runs the knowledge-rerun loop used for the DPO data. Each dataset entry contains:

- the baseline trajectory until the drifted step,
- up to three knowledge candidates generated by the loop,
- score comparisons between the baseline and every new knowledge snippet.

These runs are written to `exp8_csv_small/train_v3_dpo_3.json`.

## Generating train_v3 runs

1. Ensure `KYOTOAI_API_KEY` and the LLM credentials are exported (see the root `README.md` for details).
2. Run the training script:

   ```bash
   python exp8_csv_small/train_v3.py \
     --runs-dir seimei_runs \
     --exp-dir exp8_csv_small
   ```

   The command above uses the defaults baked into the script (randomized base system prompts, three knowledge iterations, etc.) and produces `train_v3_dpo_3.json`.

## Converting to `dataset_list`

Use `dpo_converter.py` to transform the tracker log into the format expected by the DPO trainer:

```bash
python dpo_converter.py \
  --input-path exp8_csv_small/train_v3_dpo_3.json \
  --output-path exp8_csv_small/dataset_list.json
```

### Converter behavior

- The converter removes the last agent step from every message history so the query mirrors the state *before* the drifted step.
- `_format_prompt` reproduces the rmsearch query block from `seimei.py` (user query, recent agent findings, and knowledge cues) and appends the candidate knowledge text + tags.
- Each knowledge entry becomes one `batch` element: `{"msg": [{"role": "user", "content": ...}]}`.
- `dpo_pairs` copies the `comparison` list emitted during training (winner index first, loser index second).

### Useful CLI arguments

All constants used by the converter can be overridden:

- `--max-query-messages`, `--max-query-chars` – control how much of the transcript feeds the rmsearch-style query block.
- `--max-agent-steps`, `--max-agent-chars` – limit how many recent agent findings are embedded.
- `--max-knowledge-per-step`, `--max-knowledge-chars` – trim knowledge cues pulled from agent logs.
- `--indent` – indentation level for the emitted `dataset_list` file.

The default output (`exp8_csv_small/dataset_list.json`) contains a Python-style assignment (`dataset_list = [...]`) so it can be imported or `exec`'d directly.
